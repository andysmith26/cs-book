Start your thinking

Decide whether you're going to work alone or with a partner.

#1. On paper write these initial ideas about your game:

  • the learning goal
  • the setting
  • the things to collect
  • how you win
  • how you lose

#2. Create a map of your world on paper or online (google slides or other drawing program).

#3. Write a list of questions you'll need to find answers to for this to work

Start your blog

Instructions for setting up your blog are here.

Create a new blog post and in it type and/or post images of #1, #2, and #3 from ideation above. See here for simple instructions about how to add an image to your blog.

Start your welcome screen

Create a new folder on your desktop or somewhere else on your computer that you can easily access.

In that folder create a new file called index.html.

In that file, paste this code:

<!DOCTYPE html>
<html>
  <head>
    <title>CYOA NAME</title>
  </head>
  <body>
    <h1>Welcome</h1>
    <p>This adventure is about ...</p>
    <p>Make sure you ...</p>
    <a href='forest.html'>click here to enter the forest and begin</a>
  </body>
</html>

Save the file, and view it in a browser.

Go back to your text editor and make some changes in the file, then save it.

Go back to your browser and refresh the page. You should see the changes you just made.

Start your game pages

Open your command line (PC) or terminal (Mac).

Use the cd command to get to your CYOA folder. Depending on how your computer is set up, it might look like this: cd desktop then press ENTER, then cd my-folder-name and ENTER. Don't hesitate to ask for a hand with this part.

Use the dir command (PC) or the ls command (Mac) to check if you are in the right folder. You should see your index.html file listed.

Make a new page by following these steps:

  1. Type cp index.html forest.html to make a duplicate file. (but replace forest.html with whatever your first location is called. Just don't forget the .html at the end! And don't use any spaces in your filename. If you want to use multiple words, you can put dashes between each word, like this: creepy-graveyard.html
  2. Type dir (PC) or ls (Mac) to see if your new file is now listed.
  3. Open your new file in a text editor and make the necessary updates. Feel free to ask for support or inspiration here!
  4. View that new page in your browser.
  5. View the welcome page in your browser and test the link to your new page.

Make more game pages

Repeat the numbered steps you just did (listed above) and adapt them as needed to build more locations for your adventure.

Test your links often!

Requirements

Every game must include all of the Basic requirements listed below, and every game must include at least one of the Advanced requirements listed below.

Basic

  1. your game has at least 5 location pages
  2. each page uses css for styling the page background and text
  3. each page uses at least one header tag and one paragraph tag
  4. at least one page uses list tags
  5. at least one page has an image on it (try out image alignment here)

Advanced (choose at least one)

None of these are easy. They are listed roughly in order from easiest to hardest, according to your teacher's totally unscientific opinions of them.

  • use a hand-drawn "image map" to control your navigation (scan your image then use this tool to create some code)
  • use css to position your elements on at least two pages (try it out here)
  • use bootstrap components on at least two pages (browse the components here) (set up your page like the starter template here)
  • use javascript to pass information between among all the pages of the game (here's how)

Publishing Your Game

Fork this repo. Then click `Upload files` and upload all the files from the `cyoa` folder on your computer. Wait a couple minutes for github to update its web servers, then see how it looks at YOURUSERNAME.github.io/cyoa (except replace YOURUSERNAME with, well, your username).

To make changes, update your file locally (which means on your machine) and then click `Upload files` again, and upload the new file. If you already have a file in your repo with the same name, Github will replace it with the new file.
