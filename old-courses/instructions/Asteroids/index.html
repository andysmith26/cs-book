Goals

  • Create a Java project using Greenfoot
  • Use if statements and predicate methods
  • Use inheritance to abstract duplicated functionality into a parent class
  • Use instance variables to keep track of the state of an object
  • Use comparison operators to detect when an object touches the edge of a screen
  • Make an original improvement to existing code
  • Communicate about the functionality and purpose of code segments
  • Identify fundamental computing concepts in existing code
  • Reflect on and communicate about the process of developing working code

Project Specifications

Your project consists of three deliverables:
  1. An Asteroids game that runs within Greenfoot.
    1. The game includes all of the elements mentioned in 4 packets (except the 2-player implementation, which is optional).
    2. The game includes at least one original improvement, designed and implemented by you with minimal troubleshooting assistance.
  2. A 15-30 second video on YouTube that shows a screen-recording of you playing your Asteroids game.
    1. The video privacy is set to unlisted or public.
    2. A link to the video is on the README page of the class repo, together with your first name.
  3. A poster meeting the following requirements.
    1. The poster is good-looking and polished, and its size is minimum 11x17 and maximum 24x36.
    2. The poster includes screenshots (at least 2, but not covering more than 50% of the poster) illustrating the improvement you made to the game.
    3. Those screenshots are annotated to explain what they illustrate and to identify the purpose of your improvement.
    4. The poster includes a paragraph (max 150 words) describing the incremental development process of your improvement.
    5. That paragraph focuses on one distinct point in that process where you encountered a difficulty (or opportunity) and how it was resolved (or incorporated).
    6. The poster includes a code segment (4-20 lines) from the Asteroids project (not necessarily from your improvement) that implements one of the concepts covered in the midterm review guide.
    7. That code segment is annotated to illustrate how it implements the main ideas of the selected concept, and how it contributes to the overall functioning of the game.

Step 2: Original Improvement

Make one original improvement to the game. (ie. welcome screen, or extra ship functionality, or...)

If you work with somebody on this part, your original improvement must be substantially different than your partner's. This means it must be *your* improvement. But if you are stuck or need troubleshooting support, feel free to talk it out with someone.


Step 3: YouTube Video

Record 15-30 seconds of your screen while playing the Asteroids game. You can do this however you want, but you must publish the video on YouTube. Instructions for a simple way to do this (with your school account, directly in YouTube) are here.

After it is published, make sure the privacy setting for your video is either 'Unlisted' or 'Public'.

Now you need to share your video on our class Asteroids repo. To do this, go the the WoodstockCS repo Asteroids (linked here). Don't make a fork of this repo. You're going to edit the class repo directly.

In order to edit this repo, you need to accept an invitation from Mr. Smith to be part of the GitHub team called '1718-CSA', which came to your email inbox on Jan 1. After accepting this invitation, go to the repo and put the README.md file in edit mode and add a line like this:

which is called 'Markdown code' and after you commit that change, it will look like this:
  Andy

After you commit the change to the README.md page, view the main repo page again and check that your link works.


Step 4: Poster

Read the Specifications above for detailed requirements related to the Poster part of this project. Don't wait until the last minute to ask questions.


Definition of Done

You are done when:

  • You have a fully functioning, polished-looking Asteroids game which you can run from within Greenfoot, and that game incorporates all of the requirements mentioned in the Specifications above.
  • The main page of the class Asteroids repo (linked here) contains a link to your YouTube video, which meets all of the requirements mentioned in the Specifications above.
  • You have a poster on hand in Mr. Smith's classroom, which meets all of the requirements mentioned in the Specifications above.
